# 스프린트 9 회고 — 2026.03.04

## 스프린트 목표
아카이브 무한스크롤, 카드 랜딩 페이지 PixelScene 배경, 모바일 키보드 레이아웃 대응, 누락 GA 이벤트 추가

## 에이전트 간 회고

### 잘된 점 (Keep)
- 커서 기반 페이지네이션을 DB → API → 컴포넌트 3계층에 걸쳐 일관되게 구현
- IntersectionObserver + sentinel 패턴으로 무한스크롤을 깔끔하게 구현 — 별도 라이브러리 없이 네이티브 API만 사용
- 카드 랜딩 페이지에 Client/Server Component 분리 패턴 적용 — Server Component의 DB 접근과 Client Component의 Canvas를 자연스럽게 조합
- visualViewport API로 키보드 감지 → 레이아웃 동적 조정 — CSS만으로는 해결 어려운 부분을 JS로 보완
- 4개 트랙(무한스크롤, PixelScene, 키보드, GA)을 한 스프린트에서 빌드 에러 없이 완료
- 애니메이션 delay에 `Math.min(index, 19)` 적용하여 무한스크롤로 추가된 카드의 stagger 지연 방지

### 개선할 점 (Problem)
- API 응답 형식이 배열에서 `{cards, nextCursor}` 객체로 변경됨 — 하위호환 고려가 필요할 수 있었으나, 현재는 ArchivePhase만 사용하므로 문제 없음
- 카드 넘버링(NO.XXX)이 totalCount 기준인데, 삭제 후 넘버링이 맞지 않을 수 있음 — 현재는 허용 범위
- visualViewport API가 일부 구형 브라우저에서 미지원 — `if (!vv) return` 가드로 대응했지만, 구형 모바일에서는 키보드 대응이 동작하지 않음

### 시도할 것 (Try)
- 무한스크롤 성능 모니터링 — 카드가 수백 개 이상일 때 DOM 노드 수 관찰
- 카드 넘버링을 서버에서 row_number로 계산하는 방안 검토
- iOS Safari에서 visualViewport 동작 실기기 테스트

### 기술 부채 메모
- PixelScene `forwardRef` + `captureImage` 여전히 미사용 (Sprint 7부터)
- `getAllCards()` 함수가 DB에 남아있으나 현재 사용처 없음 — 향후 정리 대상
- 카드가 수백 개 이상 쌓이면 DOM 가상화(virtualization) 검토 필요

### 주요 의사결정 사항
- **페이지네이션 방식**: offset 대신 커서 기반 — created_at 기준으로 안정적 정렬, 삭제 시에도 페이지 누락 없음
- **무한스크롤 vs 버튼 페이지네이션**: 무한스크롤 선택 — 갤러리/아카이브 UX에 더 적합
- **PixelScene 적용 방식**: Server Component를 유지하면서 Client wrapper로 Canvas 분리 — SEO/OG를 위한 서버 렌더링 보존
- **키보드 대응 범위**: 레이아웃 조정(justify-start) + 제목 마진 축소 + 티커 숨김 — 과도한 리레이아웃 없이 핵심만 대응
