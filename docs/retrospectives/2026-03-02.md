# 스프린트 회고 — 2026.03.02

## 에이전트 간 회고

### 잘된 점 (Keep)
- 5개 파일에 걸친 디자인 개선을 체계적 플랜 기반으로 진행
- 빌드 검증 → 커밋 → 배포 → 확인 사이클을 빠르게 반복
- PO 피드백을 즉시 반영 (글자수 카운터 제거, 버튼 문구, 테두리 복원 등)

### 개선할 점 (Problem)
- 첫 번째 일괄 적용 후 linter에 의해 전체 revert됨 → 원인 파악 없이 재적용
- 닉네임 랜덤 버튼 테두리 변경을 PO 의도 확인 없이 진행 → 즉시 복원 요청 받음
- 커밋이 8개로 분산되어 squash 필요 → 처음부터 의미 단위로 묶었으면 force push 불필요

### 시도할 것 (Try)
- 디자인 변경 시 PO 의도가 불명확하면 먼저 확인 후 적용
- 반복적인 소규모 커밋 대신, 관련 변경을 묶어서 커밋
- linter revert 발생 시 원인을 먼저 분석

### 기술 부채 메모
- 입력 필드 스타일이 인라인 style로 관리됨 → CSS 클래스로 통합 고려
- PixelScene의 모바일 분기(`isMobile`)가 초기 렌더 시점의 `window.innerWidth`에 의존 → resize 대응 부족

| 애니메이션 미세조정 (쉬머 축소, 모바일 크기) | 완료 |
| PO 추가 요청 (글자수 제거, 문구 변경, 정렬, 너비 통일) | 완료 |

### 주요 의사결정 사항
- 입력 border `2px → 1px`: 프레임 테두리와 겹침 해소 목적
- 입력 박스 `width: 98%` + 가운데 정렬: 프레임 안에서 여백 확보
- 모바일 픽셀아트 `60x24 → 50x18`, 너비 `90% → 70%`: 화면 대비 비율 축소
