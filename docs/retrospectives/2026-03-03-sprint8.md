## 에이전트 간 회고

### 잘된 점 (Keep)
- 계획 단계에서 파일 목록, 상세 구현, 검증 방법까지 명확하게 정의 → 구현 시 의사결정 지연 없음
- 새 파일 5개 + 수정 파일 5개를 병렬로 작업하여 빠르게 완료
- 기존 OG 이미지 패턴(opengraph-image.tsx)을 그대로 재사용 → 카드별 OG도 일관된 NES 스타일 유지
- Next.js 16의 `params: Promise` 패턴을 정확히 적용 (await params)
- i18n 사전에 KO/EN 동시 추가하여 향후 영어 활성화 시 추가 작업 불필요
- SELECT_ARCHIVE 액션에서 cardId 누락 발견 → 즉시 수정하여 아카이브에서 열람한 카드도 링크 복사 가능

### 개선할 점 (Problem)
- 법률 페이지 내용이 실제 법률 검토 없이 작성됨 → PO의 검토/수정 필요
- 개인정보 보호책임자 이메일을 임의로 설정 (pointlessjoy@proton.me) → PO 확인 필요
- GA 이벤트 중 `card_delete`, `admin_login`은 계획에 있었으나 ArchivePhase 수정 범위를 최소화하기 위해 이번 스프린트에서 제외

### 시도할 것 (Try)
- 법률 관련 콘텐츠는 PO와 사전 검토 후 작성
- 누락된 GA 이벤트(card_delete, admin_login) 다음 스프린트에서 추가
- 카드 랜딩 페이지에 PixelScene 배경 추가 검토 (현재 단색 배경)

### 기술 부채 메모
- ArchivePhase에 GA 이벤트 (card_delete, admin_login) 미적용
- 카드 랜딩 페이지가 Server Component라 PixelScene(Canvas) 배경 없음 — 필요 시 Client wrapper 추가
- 법률 페이지 내용 실제 법률 자문 미반영
