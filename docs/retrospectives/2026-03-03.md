## 에이전트 간 회고

### 잘된 점 (Keep)
- 4개 트랙(공유/관리자/아이콘/i18n+UX)을 하나의 스프린트에서 체계적으로 소화
- 앱 아이콘 디자인을 PO와 여러 라운드 반복하며 만족스러운 결과 도출 (별빛 + 커서 + glow/reflection)
- 공유 카드를 1080×1080 고정 사이즈로 새로 렌더링 — 화면 캡처 방식 대비 품질이 크게 개선됨
- 이전 스프린트의 기술 부채(인라인 스타일, PixelScene resize 버그, 데드코드) 모두 해소
- PO 피드백 즉시 반영 사이클 유지 (아이콘 6회 이상 반복, 공유 카드 폰트 크기 조정 등)

### 개선할 점 (Problem)
- Vercel 환경변수 설정 시 값이 올바르게 들어갔는지 검증 단계 누락 → 관리자 로그인 실패로 재설정 필요
- OG 이미지 폰트 로드 시 woff2 → woff → TTF 순으로 시행착오 — satori의 TTF only 제약을 미리 파악했으면 1회에 해결 가능
- 카카오 OG 캐시 문제를 사전에 안내하지 못함 → PO가 여러 번 확인하게 됨
- i18n 영어 버전을 구현했지만 바로 숨김 처리 — 초기에 스코프를 좁혔으면 불필요한 작업 회피 가능

### 시도할 것 (Try)
- 환경변수 설정 후 API 호출로 즉시 검증하는 단계 추가
- OG 이미지 변경 시 카카오/페이스북 캐시 초기화 안내를 선제적으로 제공
- 외부 라이브러리(satori, ImageResponse) 제약사항을 구현 전에 확인
- 스프린트 스코프가 클 때 PO와 우선순위 합의를 먼저 진행

### 기술 부채 메모
- PixelScene의 `forwardRef` + `captureImage`가 현재 미사용 상태 — 공유가 독립 렌더링으로 전환됨. 향후 필요 없으면 제거 고려
- 공유 카드에서 Galmuri 폰트는 클라이언트 로드에 의존 — 폰트 미로드 시 monospace 폴백

### 주요 의사결정 사항
- **앱 아이콘 모티프**: "깜빡이는 커서 _" — "무용한 기록"의 추상화로 PO와 합의
- **공유 카드 레이아웃**: 프레임 안에 답변(크게) + 생성 텍스트(작게), 하단에 질문 문구
- **공유 방식**: 캔버스 캡처 대신 독립 렌더링 — 해상도/품질 일관성 확보
- **i18n 영어 숨김**: 시스템은 구축하되 UI 토글은 비활성 — 향후 필요 시 즉시 활성화 가능
- **페이지 제목**: "무용한 것들의 아카이브" 검토 후 "무용한 기쁨" 유지 — 짧고 감정적 울림이 강함
- **관리자 비밀번호**: 서버 환경변수 방식, 클라이언트에 노출 없음
